Honest limitations

What this isn't.

Most landing pages skip this section. We think it's the most useful part.

  • Not a replacement for antivirus. Defender catches signature-known malware faster than we do. Run both.
  • Not an EDR. No kernel driver, no real-time blocking, no quarantine of running threats. We surface; we don't intercept.
  • Not a SIEM. We run on one PC and analyze that PC. We don't ingest from servers, cloud, or endpoints across a fleet.
  • Not a vulnerability scanner. We don't enumerate CVEs against installed software versions. Use OpenVAS or Nessus for that.
  • Not for Mac or Linux. Yet. Windows-only because that's where the malware and the buyers are.
  • Not magic. If an attacker with admin clears the audit log before our scan, we won't see what they did. We'll see the log was cleared — which is its own Critical alert.

Questions

What people actually ask.

Will this replace my antivirus?

No, and please don't try. Antivirus catches known-bad signatures fast; we watch behavior across all Windows telemetry sources. Different problems, different solutions. We even monitor whether Defender is running and warn you if it gets disabled.

Does it phone home?

No. Zero outbound packets unless you explicitly configure something (e.g., your own SMTP server for email alerts to your own address). Run Wireshark while it's open and verify yourself. Our network calls would all be visible. There aren't any.

Why one-time pricing? Everyone else does subscriptions.

Subscription fatigue is real, and our maintenance cost doesn't scale per scan. Pay once, you own the version you bought, get a year of updates, and use it forever. No "trial expired" lockouts. No "we changed the licence in v2." If we go out of business, your .exe still works.

I'm not technical. Will I get overwhelmed?

The Health Check tab is built for non-technical users. One score. Cards in plain English. "Fix it" buttons. The other 13 tabs exist when you want to dig — but you don't have to. We deliberately put the simple view first.

What if I block something I shouldn't have?

Everything is reversible. Firewall rules are tagged LogSentinel_*. Hosts-file edits are commented # LogSentinel|.... Snoozed findings, custom rules, scan history — all in ~/.log_sentinel/ as plain JSON. Audit and undo any change at any time.

Does it work without Administrator rights?

Most of it does. Processes, network, services, scheduled tasks, autoruns, USB history, scans, Live Mode, GeoIP — all work as a normal user. Some features (Security Event Log, firewall changes, hosts-file edits, panic-button isolation) need admin. We tell you clearly when something needs it.

Will antivirus flag the .exe?

Possibly, if it's not yet code-signed. PyInstaller binaries are sometimes flagged for being PyInstaller binaries — heuristic false positives. Once we're signed by a CA-issued cert, this stops. The source is open if you want to build it yourself.

Where does my data live?

On your computer, in ~/.log_sentinel/. Your scan history, custom rules, preferences, snooze list, FIM baseline, honeypot list, email config — all there. Delete that folder and Log Sentinel forgets everything.
