# Portfolio Summary

## Log Sentinel

Log Sentinel is a Windows-focused security and system health tool built for local endpoint review, SOC practice, and support reporting.

## What It Shows

- Windows Event Log collection and parsing
- Failed-login and authentication event review
- Host telemetry collection from processes, services, tasks, autoruns, DNS, USB history, software inventory, and network connections
- Rule-based detection engineering
- MITRE ATT&CK mapping
- Severity scoring and analyst-style prioritization
- Plain-English remediation guidance
- HTML/PDF reporting
- Testable demo detections using harmless local artifacts

## Analyst Workflow

1. Run Log Sentinel as Administrator.
2. Collect Security, System, and Application logs.
3. Review the Health Check for prioritized issues.
4. Open Logs to inspect raw authentication and system events.
5. Review Findings for mapped detections and recommended actions.
6. Use Timeline to understand when activity occurred.
7. Export a report for evidence, support, or portfolio documentation.

## Why It Matters

This project turns manual Windows triage tasks into a repeatable analyst workflow. It is useful for demonstrating practical understanding of host-based investigation, Windows logging, detection logic, and communicating findings to non-technical users.
